# Message Broker

Реализация тестового задания на Go: HTTP-сервис с очередями сообщений.

Сервис принимает сообщения в именованные очереди и отдаёт их в порядке FIFO.  
Если очередь пуста, `GET`-запрос может ждать сообщение до истечения таймаута.

## Задание

Нужно реализовать брокер очередей в виде веб-сервиса.

### 1. `PUT /queue?v=message`

Положить сообщение `message` в очередь с именем `queue`.

Примеры:

```bash
curl -X PUT 'http://127.0.0.1:4000/pet?v=cat'
curl -X PUT 'http://127.0.0.1:4000/pet?v=dog'
curl -X PUT 'http://127.0.0.1:4000/role?v=manager'
curl -X PUT 'http://127.0.0.1:4000/role?v=executive'
```

Ответ:

- пустое тело
- статус `200 OK`

Если параметр `v` отсутствует:

- пустое тело
- статус `400 Bad Request`

### 2. `GET /queue`

Забрать сообщение из очереди `queue` по принципу FIFO.

Примеры:

```bash
curl 'http://127.0.0.1:4000/pet'        # cat
curl 'http://127.0.0.1:4000/pet'        # dog
curl 'http://127.0.0.1:4000/pet'        # 404
curl 'http://127.0.0.1:4000/role'       # manager
curl 'http://127.0.0.1:4000/role'       # executive
curl 'http://127.0.0.1:4000/role'       # 404
```

Для `GET` можно передавать аргумент `timeout`:

```bash
curl 'http://127.0.0.1:4000/pet?timeout=5'
```

Если в очереди нет готового сообщения, получатель ждёт:

- либо появления сообщения,
- либо истечения таймаута.

Если сообщение так и не пришло, нужно вернуть:

- пустое тело
- статус `404 Not Found`

### Дополнительные требования

- Получатели должны получать сообщения в том же порядке, в котором пришли запросы.
- Если два получателя ждут сообщения, первое сообщение получает тот, кто запросил раньше.
- Порт сервиса задаётся через аргументы командной строки.
- Запрещено использовать сторонние пакеты, только стандартную библиотеку Go.
- Желательно держать решение в одном `.go`-файле.

## Запуск

```bash
go run . 8080
```

Или после сборки:

```bash
go build -o message_broker .
./message_broker 8080
```

## Запуск в Docker

Сборка образа:

```bash
docker build -t message_broker .
```

Запуск контейнера на порту `8080`:

```bash
docker run --rm -p 8080:8080 message_broker 8080
```

Если нужен другой порт, передай его вторым аргументом:

```bash
docker run --rm -p 9000:9000 message_broker 9000
```
